kuangbin匹配专题-B

2019-04-05
  • 二分匹配

HDU - 1281
这是典型的二分图判断+二分匹配
我们考虑如何判断是否是二分图？
非常简单，我们把图进行染色，相邻的点不能为同一颜色，这样就能判断是否是二分图（多亏前几天CF）
当然需要注意的是，虽然题目要求的,能否把n个学生分成两部分，每部分内的学生互不认识，而两部分之间的学生认识。如果可以分成两部分，就算出房间最多需要多少间，否则就输出No。我们仍然对认识的进行匹配，那么最大次数=最大匹配/2

kuangbin匹配专题-A

2019-04-04
  • 二分匹配

这道题还真不容易看出来是二分图匹配。
题意很简单，给出N * N 矩阵，有一些墙，墙的位置放碉堡，其他的地方可以
但是有条件，碉堡不能在同一行或者同一列上，但是他们中间有墙的话，就可以。
问最多放置多少个碉堡。

考虑如果某个位置是能放置碉堡的，那么这个位置的横坐标和纵坐标是相互匹配的。
那么如果有碉堡呢？？？发现，那么这个位置的横纵坐标可以对应多个纵坐标，并且多个纵坐标，可以对应多个横坐标。
我们如何改成一一对应关系呢？？？
把坐标编号即可。考虑纵列，我们每遇到’x’，或者到了新的纵列，就把横坐标编号++。
那么对应纵坐标来说，不同的横坐标编号，一定是相互不能攻击到的。
同理对应横坐标也是如此。
实质就是对应某一行，其中碉堡的个数觉得了，这一行对应的列，可以匹配多少个。
然后按照编号建边，跑二分图匹配即可

cf 550 div3 Graph Without Long Directed Paths

2019-04-04
  • 树上染色
  • 二分匹配

https://codeforces.com/contest/1144/problem/F
这个题目的意思开始我并没有理解，什么叫已经走过的路径次数。后面仔细一想，的确是这样的。
已经走过的路径次数不能超过1，代表从每个点出发，最多只能往下走一个节点。
那么其实这就是一个比较裸的染色问题。
把这个点开始的下一个节点染成不同颜色（就两张颜色）
那么如果其中碰到相同颜色的节点，代表无法染色，也就是说没法染成新的颜色，那么也就是说，肯定会有路径次数大于1的出现
最后把条边指向单一颜色即可。
